# 开发日志 - 2025年9月15日

## 修复内容概览
今天主要解决了首页生成内容后分享功能的两个核心问题：
1. **分享卡片标题显示问题** - 显示"提示词"而非具体用户输入
2. **分享链接访问问题** - 好友无法正常访问分享的内容

## 问题分析与解决过程

### 问题1：分享标题显示"提示词"而非具体内容

**问题现象：**
- 微信分享卡片显示："分享的灵感 - 提示词 - 通用模型生文（有趣）"
- 应该显示具体的用户输入内容

**根本原因：**
首页分享逻辑中，`userPrompt` 获取顺序不正确：
```javascript
// 问题代码
const userPrompt = this.data.userInput || '提示词';

// 修复后
const userPrompt = inputText || this.data.inputText || this.data.userInput || '提示词';
```

**解决方案：**
修改了两个分享函数中的用户输入获取逻辑：
- `onShareAppMessage`（分享给朋友）：优先使用 dataset 中的 `inputText`
- `onShareTimeline`（分享到朋友圈）：优先使用 `this.data.inputText`

**涉及文件：**
- `/pages/index/index.js:2383` - 朋友分享
- `/pages/index/index.js:2475` - 朋友圈分享

### 问题2：分享链接访问失败 - ID关联问题

**问题现象：**
- 首页生成内容后能成功保存到历史记录
- 但分享链接使用的是临时ID，好友点击后显示"分享的内容暂时无法访问，可能是链接已过期或内容已被删除"

**根本原因分析：**
```
流程问题：
1. 首页生成 → 获得临时ID (如: 1657)
2. 保存到历史记录 → 后端生成真实的 prompt_id 和 share_id
3. 分享时仍使用临时ID → 导致访问失败

正确流程应该是：
1. 首页生成 → 获得临时ID
2. 保存到历史记录 → 获取真实ID
3. 分享时使用真实的 share_id → 成功访问
```

**详细解决步骤：**

#### Step 1: 修复ID获取逻辑
修改 `saveHistoryRecord` 函数，确保获取历史记录中的真实ID：

```javascript
// 修复前：只做标记，不获取真实ID
saveHistoryRecord: function(promptId) {
  wx.setStorageSync('needRefreshHistory', true);
}

// 修复后：主动获取历史记录的真实ID
saveHistoryRecord: function(promptId) {
  wx.request({
    url: 'https://www.duyueai.com/history',
    method: 'GET',
    data: { openid: app.globalData.openid, page: 1, limit: 1 },
    success: (res) => {
      const latestRecord = res.data.data[0];
      const realShareId = latestRecord.share_id || latestRecord.shareId || latestRecord.prompt_id;

      this.setData({
        currentPromptId: latestRecord.prompt_id,
        currentShareId: realShareId
      });
    }
  });
}
```

#### Step 2: 修复分享按钮数据绑定
WXML模板中分享按钮使用错误的ID：

```xml
<!-- 修复前 -->
<button data-share-id="{{currentPromptId}}" open-type="share">

<!-- 修复后 -->
<button data-share-id="{{currentShareId}}" open-type="share">
```

#### Step 3: 完善分享逻辑
确保分享时优先使用真实的 share_id：

```javascript
// 优先级：dataset.shareId → currentShareId → currentPromptId
const shareId = dataset.shareId || this.data.currentShareId || this.data.currentPromptId;
```

#### Step 4: 添加状态重置
确保每次生成新内容时正确清空之前的ID：

```javascript
// 在生成开始时重置状态
this.setData({
  currentPromptId: null,
  currentShareId: null,  // 新增
  isFavorited: false,
});
```
